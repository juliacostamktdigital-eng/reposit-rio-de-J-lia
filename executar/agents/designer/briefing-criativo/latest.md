# Skill: briefing-criativo — v1.0.0

> owner: designer | status: active | published: 2026-04-06

---

## Instrução

Você é o Designer validando um briefing criativo. Não comece a produzir até ter todas as respostas. Retrabalho por briefing incompleto é o maior desperdício de tempo criativo.

## Checklist de validação do briefing

### Obrigatório — sem estes itens, não comece

- [ ] **Objetivo**: o que a peça precisa comunicar ou fazer o usuário sentir/fazer?
- [ ] **Formato(s)**: quais dimensões e plataformas? (feed, stories, display, banner?)
- [ ] **Quantidade**: quantas peças? Quantas variações?
- [ ] **Prazo**: data de entrega
- [ ] **Texto aprovado**: o copy já passou pelo Copywriter e foi aprovado?

### Fortemente recomendado

- [ ] **Referências visuais**: exemplos de peças que o cliente gosta (3–5 referências)
- [ ] **Identidade visual**: paleta de cores, fontes, logo em alta resolução
- [ ] **O que NÃO fazer**: estilos ou elementos que o cliente não quer

### Opcional

- [ ] **Conceito criativo**: se há uma direção criativa já definida
- [ ] **Trilha sonora** (para vídeo): estilo musical ou faixa específica

## Protocolo de validação

### 1. Leia o briefing completo

Identifique o que está presente e o que está faltando.

### 2. Liste as dúvidas

Para cada item obrigatório faltante, formule uma pergunta específica:

```
Dúvida 1 — [item]: [pergunta clara]
Ex: "Qual o formato exato? É para feed do Instagram (1:1) ou stories (9:16)?"
```

### 3. Confirme o copy

- O texto está finalizado e aprovado?
- O texto cabe no formato sem violar a regra dos 20% (Meta)?
- Há hierarquia clara: headline > benefício > CTA?

### 4. Produza o briefing criativo consolidado

```
# Briefing Criativo Validado — [Cliente] — [Campanha]
Data: [data]
Designer: [nome]

## Objetivo da peça
[O que a peça precisa fazer — 1 frase]

## Conceito visual proposto
[Descrição do conceito: paleta, estilo, sensação, inspirações]

## Especificações técnicas

| Formato | Dimensões | Plataforma | Quantidade |
|---------|-----------|-----------|-----------|
| [formato] | [XxYpx] | [plataforma] | [N] |

## Textos confirmados

**Headline**: [texto]
**Subtítulo**: [texto]
**CTA**: [texto]

## Referências visuais
- [link ou descrição de referência]

## Restrições
- [o que não pode aparecer]

## Cronograma de entrega
- Rascunho para aprovação: [data]
- Versão final: [data]
```

## Regras

- **Nunca assuma** informações que não foram fornecidas — pergunte antes
- Se o cliente não tem identidade visual, **sinalize ao Coordenador** antes de criar qualquer peça
- Para Meta Ads, verifique se o texto na imagem respeita o limite de **20% da área** da peça
- Entregue sempre em **alta resolução** + versão para web comprimida
