# Referência Do Handoff Copy Para Vendas

Fonte auxiliar: `assets/canonicos/04_DOCUMENTO_COPY_FINAL_DCC_CANONICO.md`.

## Princípio

O handoff de copy para vendas transforma campanha em contexto comercial. Ele evita que o vendedor trate um lead gerado por uma promessa específica como se fosse um contato genérico.

## Conteúdo Obrigatório

O handoff deve incluir:

- resumo da promessa da campanha;
- expectativa criada no lead;
- principais dores do lead;
- objeções esperadas;
- perguntas de qualificação;
- script de primeiro contato;
- critérios de MQL/SQL;
- motivos de desqualificação;
- próximos passos recomendados.

## Promessa Da Campanha

Descrever:

- qual problema a campanha nomeou;
- qual transformação foi prometida;
- qual mecanismo foi apresentado;
- qual prova foi usada;
- qual CTA levou à conversão;
- qual expectativa o lead pode ter.

## Dor, Desejo E Objeção

Para cada persona:

- dor explícita;
- dor latente;
- implicação financeira, operacional ou emocional;
- desejo racional;
- desejo emocional;
- objeção esperada;
- resposta segura;
- prova que convence.

## Perguntas De Qualificação

Perguntas devem conectar dor e fit comercial.

Boas perguntas:

- investigam situação atual;
- medem urgência;
- identificam impacto;
- testam autoridade;
- qualificam timing;
- validam se a promessa da campanha faz sentido.

Evite perguntas genéricas sem ligação com a campanha.

## Script Inicial

Estrutura recomendada:

```text
Abertura contextual:
Vi que você se interessou por [oferta/campanha] sobre [dor/tema].

Conexão com promessa:
Normalmente esse tema aparece quando [situação/problema].

Pergunta de diagnóstico:
Hoje, como vocês lidam com [dor operacional]?

Aprofundamento:
Isso impacta [métrica/risco/processo] de alguma forma?

Ponte:
Se fizer sentido, posso te mostrar como o diagnóstico identifica [ponto crítico].

Próximo passo:
Vamos marcar [ação] para [objetivo concreto]?
```

## Critérios MQL/SQL

MQL deve considerar:

- persona ou fit;
- dor declarada;
- interesse no tema;
- origem/ângulo;
- capacidade de compra ou uso;
- qualidade dos dados.

SQL deve considerar:

- problema reconhecido;
- urgência;
- autoridade ou influência;
- orçamento ou potencial;
- timing;
- próximo passo aceito.

## Motivos De Desqualificação

- fora do ICP;
- sem dor relevante;
- sem autoridade/influência;
- timing incompatível;
- orçamento inviável;
- busca por algo que a oferta não entrega;
- dados inválidos;
- origem/tracking inconsistente.

## Feedback Loop Para Marketing

Vendas deve devolver:

- objeções novas;
- claims confusos;
- promessa desalinhada;
- dores que aparecem com frequência;
- perfis que convertem mas não qualificam;
- perguntas recorrentes;
- palavras reais usadas pelos leads;
- sinais de etapa de consciência.

Esse feedback atualiza biblioteca de hooks, copy, LP, plano de mídia e próximos testes.
