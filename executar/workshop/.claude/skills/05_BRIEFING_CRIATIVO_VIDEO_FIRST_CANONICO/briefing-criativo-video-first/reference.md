# Referência Do Briefing Criativo Video-First

Fonte normativa: `assets/canonicos/05_BRIEFING_CRIATIVO_VIDEO_FIRST_CANONICO.md`.

## Princípio

Criativo não é só entrega visual. Criativo é uma hipótese de aquisição empacotada em mídia.

Cada criativo deve responder:

- para quem estamos falando;
- qual dor/desejo estamos ativando;
- qual crença queremos mudar;
- qual promessa estamos fazendo;
- qual prova sustenta;
- qual ação esperamos;
- qual atributo será analisado depois.

## Estruturas Narrativas

### Vídeo

```text
Hook inicial:
Contexto:
Tensão:
Mecanismo:
Prova:
CTA:
```

Para vídeos curtos:

```text
0-3s: hook
3-10s: dor/tensão
10-20s: mecanismo/prova
20-30s: CTA
```

### Carrossel Ou Sequencial

Cada slide deve ter:

- função narrativa;
- texto principal;
- prova ou exemplo;
- visual sugerido;
- CTA ou transição;
- atributo de análise.

Funções possíveis:

- interrupção;
- identificação da dor;
- amplificação do impacto;
- exemplo concreto;
- quebra de crença;
- mecanismo;
- prova;
- convite/CTA.

Cada criativo precisa nascer com:

- `creative_id`;
- formato;
- persona;
- hook;
- dor/desejo;
- ângulo;
- etapa;
- versão;
- `utm_content` previsto.

Formato:

```text
creative_id__fmt-[formato]__icp-[persona]__hook-[hook]__mot-[motivador]__dor-[dor]__ang-[angulo]__stage-[etapa]__ver-[versao]
```

## Aprovação Interna

Um criativo só deve ser aprovado se:

- tem objetivo claro;
- tem hipótese;
- tem persona definida;
- tem hook claro;
- tem conexão com DCC/DEOC;
- tem CTA;
- respeita claims permitidos;
- tem ID;
- tem atributos para análise;
- está coerente com LP/oferta;
- tem critério de leitura.

Para carrossel:

- cada slide tem função narrativa;
- a sequência progride;
- o último slide tem CTA;
- a peça pode ser analisada por hook, problema, prova e persona;
- o conteúdo não depende de informação que só existe na apresentação original.

## Critério N2

Briefing está N2 quando:

- cada criativo tem briefing;
- briefing tem hipótese;
- briefing está conectado ao DCC/DEOC;
- briefing tem ID e parâmetros;
- aprovação está registrada;
- peça final está linkada ao briefing.

## Critério N3

Briefing está N3 quando:

- performance retroalimenta novos briefs;
- atributos vencedores são reaproveitados;
- atributos ruins viram anti-padrões;
- o time compara levas criativas.
