# Referência Do Roteiro Criativo Performance

Fonte normativa: `assets/canonicos/05_BRIEFING_CRIATIVO_VIDEO_FIRST_CANONICO.md`.

## Princípio

Roteiro de performance é a execução narrativa de uma hipótese de aquisição. Ele não pode perder persona, promessa, prova, CTA, variável testada nem tracking.

## Estrutura Video-First

Estrutura recomendada:

```text
Hook inicial:
Contexto:
Tensão:
Mecanismo:
Prova:
CTA:
```

Para vídeos curtos:

```text
0-3s: hook
3-10s: dor/tensão
10-20s: mecanismo/prova
20-30s: CTA
```

## Funções Narrativas

Use uma função por bloco/cena/slide:

- interrupção;
- identificação da dor;
- amplificação do impacto;
- exemplo concreto;
- quebra de crença;
- mecanismo;
- prova;
- convite/CTA.

## Tipos De Roteiro

### Video-Roteirizado

Bom para mensagem controlada, motion, narrativa institucional-performance e anúncios com estrutura clara.

### UGC

Bom para parecer conversa real, prova de uso, opinião ou descoberta. Deve preservar linguagem natural sem perder hipótese.

### Founder

Bom para autoridade, visão, ponto de vista e quebra de crença.

### Demo

Bom para mostrar mecanismo, fluxo, tela, antes/depois ou experiência do produto.

### Motion

Bom para conceitos abstratos, dados, processos e mensagens que dependem de síntese visual.

### Screen-Record

Bom para produto digital, dashboard, onboarding, fluxo, prova funcional ou tutorial.

### Carrossel

Bom para educação, comparação, diagnóstico, problema complexo e progressão de crença.

## Carrossel Slide A Slide

Cada slide deve ter:

- slide;
- função narrativa;
- texto principal;
- prova ou exemplo;
- visual sugerido;
- CTA ou transição;
- atributo de análise.

Progressão recomendada:

```text
Slide 1: nomear a ameaça ou oportunidade.
Slide 2: mostrar que o público tem recursos, mas falta visão/processo.
Slide 3: trazer caso ou analogia.
Slide 4: aprofundar consequência.
Slide 5: formular a nova crença.
Slide 6: apresentar mecanismo/solução.
Slide 7: CTA.
```

## Direção Visual

Defina:

- referência visual;
- elementos obrigatórios;
- elementos proibidos;
- cenário;
- pessoa/ator;
- objetos;
- textos em tela;
- ritmo;
- trilha;
- tom emocional;
- assets de marca.

Se vier de apresentação estratégica, registre:

- slide de origem;
- imagem/render/prova reaproveitada;
- elementos de KV/brandbook;
- restrições de uso;
- necessidade de redesenho para mídia;
- relação com a promessa.

## Tracking

Todo roteiro deve preservar:

- `creative_id`;
- formato;
- persona;
- hook;
- dor/desejo;
- ângulo;
- etapa;
- versão;
- `utm_content` previsto.

## QA Mínimo

Antes de entregar roteiro:

- hook aparece no início;
- cada bloco tem função;
- promessa tem prova;
- CTA é explícito;
- claim é permitido;
- a peça é produzível;
- tracking está mantido;
- variação declarada é isolada.
