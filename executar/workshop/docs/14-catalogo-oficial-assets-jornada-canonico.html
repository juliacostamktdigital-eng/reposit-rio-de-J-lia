Catálogo Oficial de Assets da Jornada

Atualizado: 2026-04-30

Fonte final: consolidação em assets/canonicos; base canônica original + canônico original sem skill merge dedicada.

Decisão de merge: sem skill dedicada; mantido como catálogo oficial e controle de duplicação de assets.

Status: v1 transversal

Escopo: lista oficial dos assets possíveis do Marketing OS / EXECUTAR

Objetivo: separar o que é etapa da jornada do que é inventário de assets. Este catálogo não é uma etapa operacional; é a referência transversal para decidir quais assets podem existir, quais são obrigatórios, quais são condicionais e quais são evidência de maturidade.

1. Regra de leitura

Nem todo asset é obrigatório para toda conta.

Cada asset deve ser classificado como:

ClassificaçãoDefinição
ObrigatórioSem ele, a jornada não roda ou não pode ser auditada como N2.
CondicionalNecessário apenas para certos cohorts, canais, mercados, tickets ou maturidades.
SuporteAjuda operação, governança ou aprendizado, mas não é entrega principal da conta.
EvidênciaProva que uma etapa foi executada ou que um componente existe.
LegadoExemplo anterior incorporado por um asset mais completo.

13. O que está fora da jornada principal

Os itens abaixo ficam fora da jornada principal porque são notas de leitura, não passos operacionais:

  • notação N1/N2/N3;
  • catálogo oficial de assets;
  • cohort x segmentos Leadster.

Contrato de dados não fica aqui: ele é etapa da jornada, porque precisa ser implementado antes de rodar campanha.

14. Critério para não virar bullshit

Um asset só entra no catálogo se responder a pelo menos uma pergunta:

  • Sem ele, o processo não roda?
  • Sem ele, não dá para auditar N2?
  • Sem ele, não dá para aprender em N3?
  • Sem ele, o time recoleta contexto toda vez?
  • Sem ele, mídia, criativo, LP ou vendas tomam decisão no escuro?

Se a resposta for não para todas, não é asset canônico. É anotação, exemplo, referência ou lixo operacional.
