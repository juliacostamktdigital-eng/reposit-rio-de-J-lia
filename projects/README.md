# projects — espaços de trabalho por projeto

Esta pasta agrupa **trabalho operacional por iniciativa**: código, documentação de implementação, scripts, planos e ferramentas locais de cada entrega ou experimento.

## Convenção

- Cada projeto ou cliente relevante ganha uma subpasta em `projects/<nome-do-projeto>/`.
- Mantenha nesta árvore o que é **específico daquele trabalho** (app, docs, automações). As **skills canônicas** do processo continuam em `executar/` e `saber/`.

## Exemplo de referência

O diretório **`project1_example/`** é um modelo (starter “vibe coding”: documentação antes de código, espelhos de skills para agentes, app React em `app/`). Use-o como blueprint ao criar um novo espaço de projeto.

## Relação com `outputs/`

- **`projects/`** — onde se **desenvolve e documenta** o trabalho.
- **`outputs/`** — onde se depositam **resultados obtidos** (exports, relatórios finais, entregas geradas). Por projeto, crie também uma pasta em `outputs/<mesmo-slug-ou-nome>/` e um `README.md` lá, conforme `outputs/README.md`.
