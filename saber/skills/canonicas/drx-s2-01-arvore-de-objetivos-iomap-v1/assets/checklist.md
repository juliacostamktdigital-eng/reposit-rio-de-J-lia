# Checklist: Árvore de Objetivos

**Objetivo:** Conduzir o cliente na construção da Árvore de Objetivos (Goal Tree / IO Map) para: Definir claramente a Meta do Sistema, Identificar os Objetivos Intermediários, Mapear os Requisitos (condições necessárias) que sustentam o alcance da meta, Criar a base lógica para identificar a trava do sistema
**Responsável:** Consultor
**Frequência:** por evento

---

## Antes de Começar
- [ ] Projeto iniciado (pós-Kickoff)
- [ ] Demanda do cliente ainda focada em soluções

---

## Execução
- [ ] Definir o Objetivo Principal (Top Goal) através da pergunta: 'Se este projeto for um sucesso absoluto, qual resultado concreto precisa acontecer para você dizer: valeu a pena?'  > Condicional: Conduzir até chegar a uma meta mensurável, temporal e sistêmica- [ ] Registrar a meta no topo da árvore e validar com pergunta: 'Se isso acontecer, o projeto fez sentido?'- [ ] Identificar os Objetivos Intermediários através da pergunta: 'O que precisa estar acontecendo de forma consistente para que essa meta seja possível?'  > Condicional: Conduzir até chegar a estados do sistema, não ações- [ ] Construir a base da árvore identificando os Requisitos do Sistema para cada Objetivo Intermediário, através da pergunta: 'O que precisa existir para que isso seja possível?'  > Condicional: Repetir até chegar a condições fundamentais do sistema. Requisitos são estados necessários, estruturas mínimas e condições habilitadoras, não tarefas ou ações- [ ] Realizar Validação Lógica da árvore com as perguntas: '1. Se esses requisitos existirem, esses objetivos intermediários se tornam possíveis? 2. Se esses objetivos existirem, a meta se torna alcançável?'  > Condicional: Validação ocorre quando ambas respostas são afirmativas
---

## Verificação Final
- [ ] Meta do sistema validada
- [ ] 4 a 6 objetivos intermediários definidos
- [ ] Base da árvore composta por requisitos do sistema (condições necessárias)
- [ ] Clareza sobre onde a estrutura do sistema pode estar falhando
- [ ] Árvore de Objetivos visual e documentada
- [ ] ⚠️ Verificar: Ausência de ferramenta definida em todos os passos compromete a padronização do artefato visual entregue ao cliente
- [ ] ⚠️ Verificar: Condicionais nos passos 1, 3 e 4 dependem de julgamento situacional do consultor, sem critério objetivo de encerramento do loop
- [ ] ⚠️ Verificar: Sem tempo estimado por passo, sessões podem se estender sem controle, comprometendo a qualidade do diagnóstico
- [ ] ⚠️ Verificar: A validação lógica do passo 5 é subjetiva e binária — não há protocolo para o caso de resposta parcialmente afirmativa
- [ ] ⚠️ Verificar: Único ator (Consultor) sem menção ao cliente como ator formal, embora o processo dependa inteiramente de suas respostas
