# Checklist: Mapeamento do Fluxo de Receita

**Objetivo:** Garantir que o fluxo de receita seja mapeado fielmente à realidade operacional, com etapas claras, dependências explicitadas e hipóteses iniciais de restrição identificadas.
**Responsável:** Consultor DR-X
**Frequência:** Uma vez por cliente — etapa de Análise Diagnóstica do DR-X
**Tempo total estimado:** 60–80 minutos (com cliente presente)

---

## Antes de Começar
- [ ] Descrição do funil comercial atual obtida (mesmo que informal)
- [ ] Modelo de monetização e precificação compreendido
- [ ] Dados de conversão por etapa coletados (se disponíveis — registrar ausência se não)
- [ ] Processos de entrega e faturamento compreendidos
- [ ] Prazos médios levantados (vendas, entrega, cobrança, recebimento)

---

## Execução

### 1. Definição do Início e Fim do Fluxo
- [ ] Definir claramente: onde o dinheiro começa a ser gerado (ex: lead qualificado, proposta enviada)
- [ ] Definir onde a receita é efetivamente reconhecida
- [ ] Alinhar conceito de receita com o cliente ⚠️ *diferença entre emissão de nota e recebimento pode ser relevante*

### 2. Mapeamento das Etapas
- [ ] Listar **todas** as etapas do fluxo — sem otimizações, como acontece na prática:
  - [ ] Geração de demanda
  - [ ] Qualificação
  - [ ] Venda
  - [ ] Entrega
  - [ ] Faturamento
  - [ ] Recebimento
- [ ] Registrar responsável por cada etapa
- [ ] Registrar critério de passagem entre etapas ⚠️ *"quem aprova que uma etapa terminou?"*

### 3. Identificação de Dependências
- [ ] Mapear dependências entre pessoas
- [ ] Mapear dependências entre áreas
- [ ] Mapear dependências entre sistemas ou ferramentas
- [ ] Identificar pontos de espera (onde o processo fica aguardando alguém ou algo)
- [ ] Identificar pontos de retrabalho recorrente

### 4. Análise de Gargalos e Perdas
Para cada etapa principal, avaliar e registrar:
- [ ] Tempo médio de execução
- [ ] Taxa de conversão para próxima etapa (quando disponível)
- [ ] Capacidade máxima atual
- [ ] Principais falhas ou exceções recorrentes
- [ ] Onde o fluxo desacelera ou onde leads/pedidos se acumulam ⚠️ *ponto de acúmulo = candidato a restrição*

### 5. Visão Sistêmica — Lente TOC
- [ ] Responder: onde o sistema perde dinheiro (etapas com alta taxa de saída)?
- [ ] Responder: onde o dinheiro fica travado (tempo alto entre etapas)?
- [ ] Responder: qual etapa limita o resultado global se resolvida?
- [ ] Registrar hipóteses iniciais de restrição (validação na CRT) ⚠️ *mínimo 1 hipótese de restrição*

---

## Verificação Final
- [ ] Fluxo mapeado com base na realidade — não no processo desejado
- [ ] Todas as etapas com responsável identificado
- [ ] Dependências explicitadas
- [ ] Pelo menos 1 gargalo ou ponto de acúmulo identificado e discutido
- [ ] Hipóteses iniciais de restrição registradas para uso na CRT
- [ ] Registro salvo com mapa visual (esboço, foto de quadro ou documento)
